# 订阅套餐表单与展示优化 实现计划

> **目标：** 重构套餐编辑抽屉的表单布局（按类型分组、M token 单位），优化钱包页套餐卡片（API/CodingPlan 分区域、差异化显示）和购买确认弹窗（按类型展示不同信息）。
> **架构：** 纯前端变更，不涉及后端逻辑和数据库。表单数据层添加 M token ↔ 实际 token 数的双向转换，UI 层按 plan_type 条件渲染不同字段区块。
> **技术栈：** React 19, TypeScript, react-hook-form + zod, Tailwind CSS, i18next

---

## 文件结构

### 修改文件
- `web/default/src/features/subscriptions/lib/plan-form.ts` — 添加 total_amount 和 rate_limit_tokens_per_window 的 M token 双向转换
- `web/default/src/features/subscriptions/components/subscriptions-mutate-drawer.tsx` — 表单布局重构为通用字段区、固定字段区、类型专属字段区
- `web/default/src/features/wallet/components/subscription-plans-card.tsx` — 套餐卡片按 API/CodingPlan 分区域展示，差异化显示 token 信息
- `web/default/src/features/subscriptions/components/dialogs/subscription-purchase-dialog.tsx` — 购买弹窗按套餐类型差异化展示总额度和限速参数
- `web/default/src/i18n/locales/en.json` — 新增翻译 key
- `web/default/src/i18n/locales/zh.json` — 新增翻译 key

      export function formValuesToPlanPayload(values: PlanFormValues): PlanPayload {
        return {
          plan: {
            ...values,
            price_amount: Number(values.price_amount || 0),
            currency: 'USD',
            duration_value: Number(values.duration_value || 0),
            custom_seconds: Number(values.custom_seconds || 0),
            quota_reset_period: values.quota_reset_period || 'never',
            quota_reset_custom_seconds:
              values.quota_reset_period === 'custom'
                ? Number(values.quota_reset_custom_seconds || 0)
                : 0,
            sort_order: Number(values.sort_order || 0),
            max_purchase_per_user: Number(values.max_purchase_per_user || 0),
            total_amount: Math.round(Number(values.total_amount || 0) * 1_000_000),
            upgrade_group: values.upgrade_group || '',
            plan_type: values.plan_type || 'api',
            rate_limit_tokens_per_window:
              values.plan_type === 'coding_plan'
                ? Math.round(Number(values.rate_limit_tokens_per_window || 0) * 1_000_000)
                : 0,
            rate_limit_weekly_multiplier:
              values.plan_type === 'coding_plan'
                ? Number(values.rate_limit_weekly_multiplier || 0)
                : 0,
          },
        }
      }
