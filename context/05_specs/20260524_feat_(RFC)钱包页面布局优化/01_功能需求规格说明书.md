### 3.1 页面清单

> **[无变化]** 页面清单和页面形式无变更。详见基线文档。

### 3.2 页面跳转流程

> [修改] 布局描述更新：添加资金独立为全宽横栏，订阅计划下移并按类型分栏

```
[入口：侧边栏/导航 - Wallet] → [钱包页面]
                                      ├─ 余额概览卡片 → 自动加载
                                      ├─ 添加资金板块（全宽横栏）
                                      │   ├─ 在线充值 + 兑换码充值
                                      │   ├─ 点击"支付方式"按钮 → [支付确认弹窗] → 确认 → 跳转第三方支付页面
                                      │   ├─ 点击"Creem产品" → [Creem产品确认弹窗] → 确认 → 跳转 Creem 结账页面
                                      │   ├─ 兑换码输入 + 点击"Redeem" → 直接完成（不打开弹窗）
                                      │   └─ 点击"Order History"按钮 → [账单历史弹窗]
                                      ├─ 订阅套餐区域（全宽）
                                      │   ├─ 上方：当前订阅状态 + 计费偏好
                                      │   ├─ 下方左栏：API Plan 可购计划列表 → 点击"Subscribe Now" → [订阅购买弹窗（由订阅功能管理）]
                                      │   └─ 下方右栏：Coding Plan 可购计划列表 → 点击"Subscribe Now" → [订阅购买弹窗（由订阅功能管理）]
                                      └─ 推荐奖励卡片 → 点击"Transfer to Balance" → [额度转赠弹窗] → 确认 → 完成转赠
```

**流程说明：**

> [修改] 第3、4点布局相关描述更新

1. 用户通过导航菜单进入钱包页面
2. 页面加载时自动获取用户余额数据和充值配置
3. 用户在添加资金板块中选择充值金额和支付方式后，系统弹出支付确认弹窗；也可在兑换码区域输入兑换码完成充值
4. 用户确认后跳转到第三方支付页面完成支付
5. 支付完成后，第三方通过 Webhook 异步回调系统，系统更新订单状态并增加用户余额
6. 用户从第三方支付页面返回时，系统通过 `show_history` URL 参数自动打开账单历史弹窗（如 `/wallet?show_history=true`），并刷新用户余额数据
7. 用户可随时打开账单历史弹窗查看交易记录

## 4. 页面功能详细说明

### 4.1 页面 1：钱包页面

#### 4.1.1 页面概述

> [修改] 页面职责描述中布局结构更新

**页面形式：**独立页面

**页面职责：**用户资金管理的核心页面。上方展示账户余额概览（当前余额、总使用量、API请求总数）；中部第一行为添加资金板块（全宽横栏，包含在线充值表单和兑换码充值）；中部第二行为订阅计划区域（上方展示当前订阅状态和计费偏好，下方左右分栏展示 API Plan 和 Coding Plan 可购计划列表）；底部展示推荐奖励卡片（推荐链接、奖励统计、奖励转赠）。

#### 4.1.2 涉及字段

> [修改] 充值表单区域字段重命名为"添加资金区域字段"；订阅计划区域字段新增分栏说明；其余区段无变化

##### A. 钱包概览区域字段

> **[无变化]** 与基线版本一致。详见基线文档。

##### B. 添加资金区域字段

> [修改] 原"充值表单区域字段"重命名为"添加资金区域字段"（即 4.1.5 中的"添加资金板块"），字段内容不变

| 字段名称 | 字段类型 | 数据来源 | 获取时机 | 校验规则 | 默认值 | 业务含义 |
|---------|---------|---------|---------|---------|--------|--------|
| 预设金额选项 | 选择按钮组 | 系统预设（由后端 amount_options 配置） | 页面加载 | 无 | 第一个预设金额 | 快捷选择充值金额，点击即选中并计算实付金额 |
| 自定义金额 | 数字输入框 | 用户输入 | 实时输入 | >= 当前支付渠道最小充值金额；整数 | 当前支付渠道最小充值金额 | 用户自定义充值金额 |
| 支付方式 | 按钮组 | 系统预设（由后端 pay_methods 配置） | 页面加载 | 无 | 第一个可用支付方式 | 选择充值所使用的支付渠道 |
| Waffo支付方式 | 按钮组 | 系统预设（由后端 waffo_pay_methods 配置） | 页面加载 | 无 | 无 | Waffo 渠道的子支付方式选择 |
| Creem产品 | 卡片列表 | 系统预设（由后端 creem_products 配置） | 页面加载 | 无 | 无 | Creem 加密货币渠道的固定面额产品列表 |
| 兑换码 | 文本输入框 | 用户输入 | 实时输入 | 非空 | 空 | 输入兑换码进行充值 |
| 实付金额 | 标签（计算字段） | 支付渠道金额计算接口返回（POST /api/user/amount、/stripe/amount、/waffo-pancake/amount） | 充值金额或支付方式变化时 | 无 | 无 | 前端调用后端金额计算接口获取实付金额，非本地计算 |

**充值配置开关字段（由后端 TopupInfo 返回，控制页面区域可见性）：**

| 配置字段 | 类型 | 业务含义 |
|---------|------|---------|
| enable_online_topup | 布尔 | 是否启用易支付在线充值 |
| enable_stripe_topup | 布尔 | 是否启用 Stripe 信用卡充值 |
| enable_creem_topup | 布尔 | 是否启用 Creem 加密货币充值 |
| enable_waffo_topup | 布尔 | 是否启用 Waffo 充值 |
| enable_waffo_pancake_topup | 布尔 | 是否启用 Waffo Pancake 充值 |
| enable_redemption | 布尔 | 是否启用兑换码功能 |
| payment_compliance_confirmed | 布尔 | 是否已完成支付合规确认 |

##### C. 订阅计划区域字段

> [修改] 可用计划列表新增分栏展示说明：按 plan_type 区分，API Plan 展示在左栏，Coding Plan 展示在右栏；其余字段无变化

| 字段名称 | 字段类型 | 数据来源 | 获取时机 | 业务含义 |
|---------|---------|---------|---------|--------|
| 当前订阅状态 | 状态徽标 | 订阅 API | 页面加载 | 展示用户当前活跃/已过期/已取消的订阅数量 |
| 计费偏好 | 下拉选择 | 用户选择 → API 保存 | 页面加载 | 选择计费扣费优先级：订阅优先、钱包优先、仅订阅、仅钱包 |
| 订阅详情（每条） | 卡片 | 订阅 API | 页面加载 | 展示计划名称、状态、到期时间、配额用量、下次重置时间、速率限制（Coding Plan） |
| 可用计划列表 | 卡片网格 | 公开计划 API | 页面加载 | 按计划类型分栏展示可购买的 API Plan（左栏）和 Coding Plan（右栏），包含价格、有效期、配额、购买限制 |
| 购买限制状态 | 标签 | 计划配置 + 用户已购数量 | 页面加载 | 达到购买上限时按钮显示"Limit Reached"并禁用 |

##### D. 推荐奖励区域字段

> **[无变化]** 与基线版本"推荐奖励区域字段"一致。详见基线文档。

#### 4.1.3 功能与按钮

> [修改] 功能按钮的卡片上下文更新为"添加资金板块"；其余功能无变化

**用户可操作功能：**

- 选择预设金额、输入自定义金额、选择支付方式并发起支付、选择 Waffo 支付方式并发起支付、选择 Waffo Pancake 充值、选择 Creem 产品并发起支付、兑换码充值、切换计费偏好、刷新订阅状态、购买订阅计划、复制推荐链接、转赠奖励到余额
  - 以上功能均与基线版本一致，详见基线文档。

- **[修改] 打开账单历史**
  - 触发按钮："Order History" 按钮
  - 按钮级别：页面级
  - 按钮位置：添加资金板块内（右上角）
  - 触发方式：点击按钮
  - 权限要求：见 2.2 页面访问权限
  - 加载状态：无
  - 功能说明：打开账单历史弹窗

**后台自动流程：**

- 页面初始化加载、兑换码获取链接展示、支付返回后余额刷新
  - 与基线版本一致，详见基线文档。

#### 4.1.4 业务规则

> **[无变化]** 本章内容与基线版本一致，包含折扣系统、最小充值金额校验、订单过期时间、支付渠道可用性控制、支付方式选择与金额计算联动、兑换码验证、配额换算、支付合规控制共8条规则。详见基线文档。

#### 4.1.5 交互逻辑

> [修改] 布局相关交互逻辑全面更新

- 预设金额卡片采用 2 列（移动端）/ 4 列（桌面端）网格布局，选中状态的卡片有边框和背景色变化
- 自定义金额输入框旁实时显示"Amount to pay:"和计算中的骨架屏或实付金额
- Creem 产品区域以独立卡片列表形式展示，与标准支付方式区域通过分隔线区分
- 当在线充值全部关闭时，整个在线充值区域替换为 Alert 提示
- 页面加载期间，概览卡片、添加资金板块、订阅计划区域和推荐奖励卡片均显示骨架屏
- **[修改]** 添加资金板块为全宽横栏，位于余额概览下方，包含在线充值和兑换码充值。在线充值和兑换码充值通过分隔线区分
- **[修改]** 订阅计划区域位于添加资金板块下方（全宽）。上方展示当前订阅状态（My Subscriptions）和计费偏好；下方左右分栏展示可用计划列表：左栏为 API Plan，右栏为 Coding Plan（移动端 md 以下纵向堆叠）。当某类型无可用计划时，对应栏位不渲染；当 API Plan 和 Coding Plan 均无可用计划时，计划列表部分不渲染（My Subscriptions 和计费偏好仍保留展示）
- 推荐奖励卡片位于订阅套餐区域下方，采用三栏布局（左侧标题+描述、中间统计数据、右侧链接+操作按钮），移动端纵向堆叠
- 推荐奖励统计数据采用三等分网格展示（Pending/Total Earned/Invites）
- 推荐奖励区域的合规提示仅在合规未确认时显示
- 无活跃订阅时，订阅优先/仅订阅选项标注"(No Active)"，并在下方显示提示文案
